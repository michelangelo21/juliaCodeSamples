22 ## OffsetArray
23 """
24 OffsetArray(A, indices...)
25
26 Return an `AbstractArray` that shares element type and size with the first argument but
27 uses the supplied `indices` to infer its axes. If all the indices are `AbstractUnitRange`s then
28 these are directly used as the axis span along each dimension. Refer to the examples below for other
29 permissible types.
30
31 Alternatively it's possible to specify the coordinates of one corner of the array
32 and have the axes be computed automatically from the size of `A`.
33 This constructor makes it convenient to shift to
34 an arbitrary starting index along each axis, for example to a zero-based indexing scheme followed by
35 arrays in languages such as C and Python.
36 See [`Origin`](@ref) and the examples below for this usage.
37
38 # Example: offsets
39
40 There are two types of `indices`: integers and ranges-like types.
41
42 Integers are recognized as offsets, where `0` means no offsets are applied:
43
44 ```jldoctest; setup=:(using OffsetArrays)
45 julia> A = OffsetArray(reshape(1:6, 2, 3), -1, -2)
46 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 0:1, -1:1) with eltype $Int with indices 0:1×-1:1:
47 1 3 5
48 2 4 6
49
50 julia> A[0, 1]
51 5
52 ```
53
54 Examples of range-like types are: `UnitRange` (e.g, `-1:2`), `CartesianIndices`,
55 and `Colon()` (or concisely `:`). A `UnitRange` specifies the axis span along one particular dimension,
56 `CartesianIndices` specify the axis spans along multiple dimensions, and a `Colon` is a placeholder
57 that specifies that the `OffsetArray` shares its axis with its parent along that dimension.
58
59 ```jldoctest; setup=:(using OffsetArrays)
60 julia> OffsetArray(reshape(1:6, 2, 3), 0:1, -1:1)
61 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 0:1, -1:1) with eltype $Int with indices 0:1×-1:1:
62 1 3 5
63 2 4 6
64
65 julia> OffsetArray(reshape(1:6, 2, 3), :, -1:1) # : as a placeholder to indicate that no offset is to be applied to the first dimension
66 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 1:2, -1:1) with eltype $Int with indices 1:2×-1:1:
67 1 3 5
68 2 4 6
69 ```
70
71 Use `CartesianIndices` to specify the coordinates of two diagonally opposite corners:
72
73 ```jldoctest; setup=:(using OffsetArrays)
74 julia> OffsetArray(reshape(1:6, 2, 3), CartesianIndex(0, -1):CartesianIndex(1, 1))
75 2×3 OffsetArray(reshape(::UnitRange{$Int}, 2, 3), 0:1, -1:1) with eltype $Int with indices 0:1×-1:1:
76 1 3 5
77 2 4 6
78 ```
79
80 Integers and range-like types may not be combined in the same call:
81
82 ```julia
83 julia> OffsetArray(reshape(1:6, 2, 3), 0, -1:1)
84 ERROR: [...]
85 ```
86
87 # Example: origin
88
89 [`OffsetArrays.Origin`](@ref) may be used to specify the origin of the OffsetArray. The term origin here
90 refers to the corner with the lowest values of coordinates, such as the left edge for an `AbstractVector`,
91 the bottom left corner for an `AbstractMatrix` and so on. The coordinates of the origin sets the starting
92 index of the array along each dimension.
93
94 ```jldoctest; setup=:(using OffsetArrays)
95 julia> a = [1 2; 3 4];
96
97 julia> OffsetArray(a, OffsetArrays.Origin(0, 1))
98 2×2 OffsetArray(::$(Array{Int,2}), 0:1, 1:2) with eltype $Int with indices 0:1×1:2:
99 1 2
100 3 4
101
102 julia> OffsetArray(a, OffsetArrays.Origin(0)) # set the origin to zero along each dimension
103 2×2 OffsetArray(::$(Array{Int, 2}), 0:1, 0:1) with eltype $Int with indices 0:1×0:1:
104 1 2
105 3 4
106 ```

138 function overflow_check(r, offset::T) where T
139 # This gives some performance boost https://github.com/JuliaLang/julia/issues/33273
140 throw_upper_overflow_error() = throw(OverflowError("Boundary overflow detected: offset should be <= $(typemax(T) - last(r)) for offsets of type $T, received $offset"))
141 throw_lower_overflow_error() = throw(OverflowError("Boundary overflow detected: offset should be >= $(typemin(T) - first(r)) for offsets of type $T, received $offset"))
142
143 if offset > 0 && last(r) > typemax(T) - offset
144 throw_upper_overflow_error()
145 elseif offset < 0 && first(r) < typemin(T) - offset
146 throw_lower_overflow_error()
147 end
148 end

442 function _safe_searchsorted(v::OffsetArray, x, ilo::T, ihi::T, o::Base.Ordering) where T<:Integer
443 u = T(1)
444 lo = ilo - u
445 hi = ihi + u
446 @inbounds while lo < hi - u
447 m = (lo + hi) ÷ 2
448 if Base.lt(o, v[m], x)
449 lo = m
450 elseif Base.lt(o, x, v[m])
451 hi = m
452 else
453 a = searchsortedfirst(v, x, max(lo,ilo), m, o)
454 b = searchsortedlast(v, x, m, min(hi,ihi), o)
455 return a : b
456 end
457 end
458 return (lo + 1) : (hi - 1)
459 end
